# Screenshots

> These images are generated by running the app — they are not committed yet.
> Capture them after `docker compose up` (or a local run), save them into
> `docs/screenshots/`, and they will render below.

How to capture:
1. Start the app and open http://localhost:8501.
2. Register a user (the first user becomes admin), upload a sample PDF/DOCX/TXT.
3. Use each page and take a screenshot (PNG) with the filename shown below.

| Screen | File | What to show |
|--------|------|--------------|
| Login / Register | `docs/screenshots/01-auth.png` | login + create-account tabs |
| Analytics Dashboard | `docs/screenshots/02-dashboard.png` | KPI cards + the 4 charts |
| Documents | `docs/screenshots/03-documents.png` | document list with category badge |
| Upload | `docs/screenshots/04-upload.png` | file uploader + success result |
| Classify | `docs/screenshots/05-classify.png` | predicted category + probability bar chart |
| NLP | `docs/screenshots/06-nlp.png` | entities table + keyword chart + sentiment |
| GenAI | `docs/screenshots/07-genai.png` | summary / FAQ / deadlines output |
| Chat (RAG) | `docs/screenshots/08-chat.png` | answer with expandable source citations |
| Admin | `docs/screenshots/09-admin.png` | platform totals + per-user chart |
